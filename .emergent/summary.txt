<analysis>**original_problem_statement**: Build a secure, therapist-first web application for managing a therapy practice and supporting clinical decision-making.

**PRODUCT REQUIREMENTS**
- **User Roles**: Therapist, Client, and Super Admin.
- **Authentication**: Secure login for all roles. Client self-registration is disabled; clients are created by Therapists or Super Admins. Therapist registration requires Super Admin approval.
- **Practice Management**: Client profiles (editable), appointment scheduling, session notes (SOAP & DAP), secure messaging, and payment tracking.
- **Clinical Support**: Assessment suggestion engine, therapy protocol builder, and homework/resource sharing.
- **Super Admin & Subscriptions**: A Super Admin role to manage therapists (approval, suspension, manual creation, profile edits), clients (full CRUD), and a subscription module (assigning/changing plans).
- **Data Isolation**: Therapists can only see and manage their own assigned clients.
- **Subscription Expiry**: Expired/cancelled subscriptions result in read-only access for therapists, not a full block.
- **UI/UX**: Profile photo support for clients and therapists. Search functionality in admin management panels.

**User's preferred language**: English

**what currently exists?**
A full-stack application with a FastAPI backend, React frontend, and MongoDB database. Key features include:
-   **Multi-Role Authentication**: Secure login for Super Admin, Therapist, and Client roles. Client self-registration is disabled, and therapist onboarding is managed via an admin approval workflow.
-   **Super Admin Panel**: A functional dashboard for approving/managing therapists and full CRUD operations on clients. The UI for managing Subscription Plans and Coupon Codes is present but requires fixes.
-   **Therapist Dashboard**: Therapists can create, view, and edit profiles for their assigned clients. Profile photos (URL-based) are supported.
-   **Security & Access Control**:
    -   **Data Isolation**: Backend logic is in place to ensure therapists can only access their own assigned clients.
    -   **Read-Only Mode**: Therapists with expired subscriptions are switched to a read-only mode with a persistent UI banner, blocking all data modification actions.
