**Key Technical Concepts**
-   **Provider Abstraction Pattern**: Implemented a scalable architecture for external services (Email, WhatsApp) to avoid vendor lock-in, using a base class, a dynamic registry, and provider-specific adapters.
-   **Event-Driven Notifications**: Shifted from simple counters to a robust system where backend actions (e.g., creating a client, sharing a report) trigger the creation of notifications.
-   **Frontend Polling**: The  component uses a  to poll the backend periodically for new notifications, providing a near real-time experience.
-   **Dynamic Routing and UI**: The application uses URL parameters () for dynamic pages and conditional rendering extensively to tailor dashboards for different user roles and states (e.g., showing consent form vs. main dashboard).
