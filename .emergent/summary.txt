<analysis>**original_problem_statement**: Build a secure, therapist-first web application for managing a therapy practice and supporting clinical decision-making.

**PRODUCT REQUIREMENTS**
- **User Roles**: Therapist, Client, Super Admin, and Therapist Assistant.
- **Authentication**: Secure JWT-based login for all roles. Clients and Assistants are created by Therapists or Super Admins. Therapist registration requires Super Admin approval.
- **Practice Management**: Client profiles (editable), appointment scheduling (with availability, slot generation, and calendar blocking), session notes (SOAP & DAP), secure messaging, and payment tracking.
- **Clinical Support**: Assessment suggestion engine, therapy protocol builder, and homework/resource sharing.
- **Super Admin & Subscriptions**: A Super Admin role to manage therapists (approval, suspension, manual creation, profile edits), clients (full CRUD), and a subscription module (assigning/changing plans).
- **Therapist Assistant Role**: A restricted role linked to a single therapist to manage operational tasks (clients, appointments) without access to clinical data. Includes audit logging for assistant actions.
- **Data Isolation & Security**: Therapists can only see their own assigned clients. Assistants can only access their assigned therapist's data. Expired/cancelled subscriptions result in read-only access.
- **Global Standards**: The entire application must use India Standard Time (IST), DD/MM/YYYY date format, 24-hour (HH:mm) time format, and Indian Rupee (₹) currency.
- **UI/UX**: Profile photo support, search functionality, a redesigned therapist dashboard, and productivity features like Copy Day for availability.

**User's preferred language**: English

**what currently exists?**
A full-stack application with a FastAPI backend, React frontend, and MongoDB database. Key features include:
-   **Multi-Role Authentication**: Secure login for Super Admin, Therapist, Client, and the newly added Therapist Assistant role.
-   **Super Admin Panel**: A functional dashboard for approving/managing therapists, full CRUD on clients, and subscription management.
-   **Therapist Dashboard**: A redesigned overview page with at a glance stats. Therapists can manage clients, a full-featured appointment calendar (including availability settings, slot-based scheduling, and calendar blocking), create/edit session notes with templates, and manage their assistants.
-   **Therapist Assistant**: Assistants can be created and managed by therapists. They have their own dashboard with a limited sidebar and can manage clients and appointments on behalf of their assigned therapist, with all clinical data access restricted at the backend level. Basic audit logging for assistant actions is in place.
-   **Secure Messaging**: A complete and tested messaging system allowing therapists to communicate with their clients, with controls for enabling/disabling messaging and respecting read-only mode.
-   **Global Standards**: An effort is underway to standardize timezone, date/time formats, and currency across the app. A utility file () has been created, and many frontend components have been modified.
